## 目標 / 非目標

**目標**：忠實復刻參考影片的兩段升階儀式感（軌道彗星 halo＋金框典藏卡），同時守住
現有「forward-only、零 Timer、pumpAndSettle 必收斂、reduce-motion 露 hero」鐵則。

**非目標**：
- 不動計費／網路／quota／RPC／migration／Edge。儀式純由 `drawStatus` 狀態機驅動。
- 不引入 Lottie/Rive/序列幀（毀確定性與測試安全，僅當理論 fallback）。
- 不做「換裝」（每 profile 僅一張 bundled 照片）。

## 已定案的關鍵決策（brainstorming）

| 主題 | 決策 |
|---|---|
| **儀式長度** | 每次（含換一位）都走**完整** ~7.5s，**不做手動跳過**。理由：換一位對 Starter/Essential 額度用完後**真的扣費**，付費動作該拿完整 payoff；重複疲乏被額度天然限速。 |
| **逃生閥** | **reduce-motion 就是逃生閥**（reveal 當下直接露 hero）。不另做手動 skip，避免污染 forward-only 鐵則、少一塊測試面。 |
| **不灌水到 10s** | 參考片 8.7→10s 是「落回 App 個人卡」轉場；我們那段只是 overlay 淡出露出**早已存在**的 hero，不補時間。有效 beat ≈ 7–8s。 |
| **兩段照片** | **同照片、靠「卡框處理」區分**：Stage-1 白/粉素框（預覽感）、Stage-2 金框＋深色玻璃資訊欄＋teal accent。不靠換裝。 |
| **附加細節** | 納入：能量邊框 青→金 描邊掃動＋噴火花、星塵加密＋光束橫掃。**不納入**：紫 3D 立方 logo（保留現金圓盤＋sparkle，on-brand 且省工）。 |

## §1 時間軸與狀態骨架（最高風險）

**保留不動**：`_intro`（卡背淡入 520ms）、`_waiting`（等 server 蓄力微動，repeat 嚴格 gate）。

**退役 `_flip`，換單一 forward-only `_reveal`（~7.5s，0→1）＋具名 beat 常數**
（沿用現有 0.48/0.86 命名切點手法）：

| 區間 | 約時長 | 內容 |
|---|---|---|
| `0 → kFlip1End` | ~0.7s | 3D 翻面 卡背→**白卡預覽**（rotateY 0→π）＋中點 flash |
| `→ kPreviewEnd` | ~1.8s | 白卡停留、資訊浮出；**屏息**（loop 已停） |
| `→ kRechargeEnd` | ~0.6s | 翻回卡背（蓄力重啟） |
| `→ kHaloClimax` | ~1.2s | 卡背發亮、能量邊框 青→金＋火花、**軌道彗星 halo 衝到高潮** |
| `→ kGrandFlipEnd` | ~0.7s | 高潮翻面 卡背→**金框典藏卡**＋flash |
| `→ kHoldEnd` | ~1.5s | 金卡停留、深色玻璃資訊欄＋teal 滑入、halo settle |
| `→ 1` | ~0.8s | overlay 淡出，露出底下 hero |

**安全鐵則（完全守住）**：`_reveal` 單向 forward、零 `Timer`、零 `repeat()`；
`completed` → `_toHidden()`。`_waiting` 仍是唯一 repeat、仍 gate 在 drawing＋非
reduce-motion、reveal 起手即 `stop()`。故 **pumpAndSettle 必收斂、widget test 不 hang**。
三條 controller dispose 先收。

**reduce-motion**（沿用現行為）：reveal 當下 `_toHidden()` 露 hero，整條兩段儀式只
跑給開動畫的人；haptic／一次性音效照觸發，不啟動 `_waiting` loop。

## §2 新繪圖與卡面狀態

### 2.1 軌道彗星 halo（the money shot，Q1）
`_OrbitalHaloPainter(progress, intensity, half)`：
- 2–3 條軌道環，每環繞 X 軸傾斜 θ 的 3D 圓：參數化 φ∈[0,2π]，
  3D 點 `(r·cosφ, r·sinφ·cosθ, r·sinφ·sinθ)` → 透視/正交投影成 2D 橢圓。
- 彗星 = head（φ=progress·2π＋環偏移）＋ M 個遞減 alpha/半徑拖尾點假裝動態模糊。
- **景深靠兩層 CustomPaint 夾卡片**：卡片下方畫 `half=back`（投影 z<0 的點）、
  卡片上方畫 `half=front`（z>0）。卡片本身 rotateY 仍用 Matrix4。
- 確定性、零 Random、`shouldRepaint` 看 progress/intensity。
- 退役 `_SweepGlowPainter`（或降級當 fallback）。

### 2.2 金框典藏卡（改造 `_CeremonyCardFront` 成兩態）
- **Stage-1 白卡預覽** = 現有白/粉 matte 卡幾乎原樣複用（`variant: preview`）。
- **Stage-2 金框典藏卡** = `variant: grand`：金框＋底部 frosted 深色玻璃資訊欄＋
  teal accent 文字。同照片、同欄位（名字／年齡·城市·職業），只換框與資訊欄處理。
  仍避開 hero 的「名字，年齡」精確字串。

### 2.3 能量邊框＋星塵（已選入）
- `_EnergyBorderPainter`：卡框 青→金 描邊掃動＋底邊噴火花（確定性粒子，
  golden-angle 佈點），只在蓄力段（kRecharge→kHaloClimax）亮。
- `_StarfieldPainter` 調密度＋加一道橫掃光束，沿用現有確定性佈點。

> **註**：teal 是儀式新引入的 accent（現 palette 為紫＋金）。依參考片納入，
> Eric 可於目檢時否決退回純金。

## §3 音效

維持離散 + 加 `riser`/`settle` 兩顆 wav（復用 `PracticeDrawSfx` 抽象＋spy＋lazy/guarded）。
- `reveal_chime` 改配 **Stage-1 白卡預覽** 揭曉。
- **屏息靠「reveal 起手提早停 loop」**（waiting_loop 必須留——等 server 變長度，
  烤死連續 bed 會 desync）。
- `riser` 在跨 `kRechargeEnd` 觸發（對齊 halo 高潮）；`settle` 在跨 `kGrandFlipEnd` 觸發。
- 觸發機制：`_reveal` listener **edge-detect 跨 beat 門檻**，每 beat 一個 idempotent
  bool，`forward(from:0)` 時全部重置 → 零 Timer、spy 可驗、pumpAndSettle 跨完所有
  門檻各觸發一次。
