# 社交知識整合設計：七步聊天法 → VibeSync

狀態：設計文件，尚未實作  
日期：2026-07-08  
範圍：Opener、analyze-chat、AI 實戰練習室、Coach 1:1 / follow-up、後續 Learning tab  
決策：萃取節奏與判斷，不照搬話術；所有使用者可見輸出必須保持尊重、真誠、可退出、非操控。

## 0. 定位

這份「七步聊天法」資料不適合直接塞進 prompt，因為原文同時混有三種東西：

1. 可用的聊天節奏：破冰、延伸、展示生活、互相篩選、建立安全感、順勢邀約。
2. 可轉譯的教學語言：狀態 + 感受、聊她 / 聊我 / 聊我們、說話留空間、邀約不是請求批准。
3. 不適合產品化的話術與框架：物化、性暗示、上下位操控、把對方當成待攻克目標。

VibeSync 要吸收的是第一、第二類；第三類只能作為「不要出現在產品裡」的負面樣本。

產品上的長期方向是：讓使用者變成更穩、更好聊、更有生活感、更懂得照顧對方安全感的人，而不是學一套壓迫式話術。

## 1. 安全轉譯規則

### 1.1 可以直接產品化

這些概念可以進入 prompt、分析、提示、拆解卡或教學卡：

- 聊她 / 聊我 / 聊我們：回覆不要只查戶口，也不要只講自己，要讓話題有來回感。
- 狀態 + 感受：比單純陳述事實更有畫面，也更容易接話。
- 說話留空間：不要把結論講死，保留讓對方補充、好奇、反問的空間。
- 生活樣本：用自然的生活片段呈現吸引力，不用自誇。
- 互相合適度：不是考核對方，而是看彼此節奏、價值觀、生活方式能不能接上。
- 輕鬆張力：幽默、吐槽、反差、角色感，但要能退，不能羞辱或逼迫。
- 安全感鋪墊：讓對方知道邀約是低壓、可拒絕、可改期、不是被逼表態。
- 順勢邀約：邀約是分享一個自然選項，不是求人批准。

### 1.2 只能內部轉譯，不可原詞外露

這些概念有教學價值，但原詞容易太遊戲化或操控化，必須換成中性說法：

| 原始概念 | 產品內安全說法 | 使用原則 |
| --- | --- | --- |
| DHV / 展示高價值 | 生活樣本、生活吸引力 | 展示具體生活，不吹噓身價或地位。 |
| 篩選 / 框架 | 互相合適度、節奏感 | 用來判斷彼此是否合拍，不用來壓低對方。 |
| 推拉 | 輕鬆張力、幽默反差 | 只做玩笑和反差，不做貶低或情緒操控。 |
| 角色扮演 | 共同小劇場、玩笑框架 | 保持可退出，不使用性暗示或權力壓迫。 |
| 可得性 | 安全感釋放、低壓選項 | 表達好感與可接近感，但不黏、不求。 |

### 1.3 明確剔除

以下類型不得進入 prompt、使用者可見文案、測試 golden output，也不應作為建議範例：

- 物化女性或把人比成資產、職位、獎品的說法。
- 明示或暗示「收割」「攻克」「控制」「讓對方合理化」的操控框架。
- 性化制服、角色權力差、羞辱式玩笑。
- 「申請入職」「哥哥引領」「老司機名車」這類上下位或物化語言。
- 對未明確同意的親密升級、過夜暗示、壓迫式邀約。

這些詞可以出現在設計文件的剔除清單中，用來建立測試黑名單；不能出現在 runtime prompt 或 App 產出裡。

## 3. 功能整合方案

### 3.1 Opener

目標：讓開場不只會冷讀，也能根據使用者想呈現的角度產生更自然的入口。

建議補強：

- 在 opener reasoning 中加入三角度：聊她、聊我、聊我們。
- 加入「狀態 + 感受」微公式：不是「我在咖啡廳」，而是「我剛從一個很吵的咖啡廳逃出來，突然覺得安靜好珍貴」。
- 補平台轉移場景：從交友軟體轉 LINE / IG 後，對方熱度下降時，開場要能接住前文，不像重開一局。
- 產出仍不能顯示內部術語；使用者看到的是自然可發送句子與簡短理由。

推薦第一批實作，因為改動集中在 `OPENER_PROMPT` 與既有 prompt contract test，沒有資料結構風險。

### 3.2 Analyze-chat

目標：讓聊天分析能指出「為什麼看似沒錯，但對方就是不升溫」。

新增診斷維度：

- 查戶口：一直問資訊，沒有情緒或觀察。
- 接話接死：回覆正確但沒有球，對方不知道怎麼接。
- 只展示不留球：講了自己的生活或價值，但沒有讓對方參與。
- 高吸引、低可接近：看起來條件很好，但對方感覺自己只是觀眾。
- 邀約像請求批准：把壓力丟給對方，而不是分享一個低壓選項。
- 安全感不足：邀約沒有時間、地點、退出空間或替代選項。

可見術語建議：

- 生活樣本
- 互相合適度
- 輕鬆張力
- 安全感鋪墊
- 順勢邀約
- 給她一顆球
- 狀態 + 感受

注意：`analyze-chat` 既有技術詞表有上限與 App Review 安全要求，實作時需要決定哪些詞真的進可見詞表，哪些只留在 scenario guidance。

### 3.3 AI 實戰練習室

目標：讓練習室不只是「升溫數字」，而是能訓練真實聊天節奏。

Hint 補強：

- 根據當前回合判斷要「聊她 / 聊我 / 聊我們」。
- 當使用者過度查戶口時，提示補一點自己的狀態或感受。
- 當使用者只講自己時，提示給對方一顆球。
- 當關係分數接近邀約門檻時，提示低壓邀約，不要硬衝。

Debrief 補強：

- 拆出本輪主要卡點：資訊太多、情緒太少；展示太多、可接近感不足；邀約壓力太高。
- 把 dateChance / invite readiness 的原因講成人話：她現在願意出來，不只是分數夠，而是因為安全感、節奏、生活連結都到位。
- 高分關係不等於立刻親密升級；85+ 的語氣可以像女友，但仍要保留界線與情境合理性。

Simulator 補強：

- 對自然生活樣本有更正向反應。
- 對空泛自誇、壓迫邀約、只丟問題有更冷的反應。
- 對安全感鋪墊與順勢邀約給出可感知的升溫。

這批屬於高風險 prompt 行為區，必須搭配 Deno test、anchor smoke、Codex 雙審後才說可以 dogfood。

### 3.4 Coach 1:1 / Follow-up

目標：保留 Coach 的真誠流，不把它變成話術訓練。

Coach 1:1 建議：

- 用「生活素材盤點」幫使用者建立可聊內容：工作節奏、興趣、休息方式、朋友互動、最近的小決定。
- 幫使用者把「我很有價值」改成「我有生活，我也願意讓你靠近一點」。
- 不使用推拉、框架、篩選等遊戲化詞彙。

Follow-up prepareInvite 建議：

- 加入「邀約是分享，不是請求批准」。
- 檢查三件事：對方有回應鉤子、邀約有具體但低壓的選項、對方有拒絕或改期空間。
- 產生的邀約要像生活提案：例如「我週六下午會去那間展，妳如果剛好想透氣可以一起走走」，而不是「妳要不要跟我出去」。

### 3.5 Learning tab（後續）

可做成短卡，不急著進第一批：

- 怎麼避免查戶口。
- 什麼是狀態 + 感受。
- 怎麼給對方一顆球。
- 生活樣本不是炫耀。
- 邀約是分享，不是請求批准。
- 如何讓好感有安全感。
- 關係升溫不是逼近，而是讓彼此更自然。

## 4. 建議實作批次

### Batch A：Opener 微公式

內容：聊她 / 聊我 / 聊我們、狀態 + 感受、平台轉移冷卻場景。  
風險：低到中；主要是 prompt 品質。  
測試：opener prompt contract、禁止詞掃描、3 組 anchor smoke。

### Batch B：Follow-up prepareInvite

內容：邀約是分享、安全感鋪墊、低壓選項。  
風險：中；會影響約會建議語氣。  
測試：prepareInvite prompt contract、拒絕壓迫式邀約 golden。

### Batch C：Analyze-chat 診斷詞補強

內容：查戶口、接話接死、只展示不留球、高吸引低可接近、順勢邀約。  
風險：高；會影響核心分析輸出與 App Review 感知。  
測試：schema contract、詞表上限、黑名單掃描、短對話 anchor smoke。

### Batch D：AI 實戰練習室 hint / debrief 對齊

內容：把新語言接到 hint、debrief、邀約成熟度與關係分數說明。  
風險：高；涉及 AI prompt、教學品質、練習體感。  
測試：Deno practice-chat、固定 persona 的 debrief golden、真機 smoke。

### Batch E：Coach 生活素材盤點

內容：把自我建設轉成可聊素材，不碰遊戲術語。  
風險：中；需要保持 Coach 1:1 的真誠定位。  
測試：coach prompt contract、情緒安全 regression。

推薦順序：A → B → C → D → E。  
理由：先補最小且最容易感知的「會開、會約」；再補核心分析；最後讓實戰練習室與 Coach 全面對齊。

## 5. 驗證與安全門檻

每個實作批次都應過以下 gate：

1. Prompt contract test：新增概念必須出現在允許位置，且不能外露內部術語。
2. 黑名單掃描：runtime prompt 與 golden output 不得包含物化、性化、操控語言。
3. Anchor smoke：
   - Opener：平台轉移後冷掉，產出自然續聊。
   - Analyze-chat：使用者展示很多但沒給球，能指出「只展示不留球」。
   - Follow-up：準備邀約時，能改成低壓生活提案。
   - Practice-chat：高關係分數時，debrief 能說明安全感與邀約成熟度。
4. Codex 雙審：任何 prompt / Edge function 實作完成後，需獨立 review 再說可以 build dogfood。
5. Eric / Bruce 真機 smoke：確認語氣不是油、不是教壞、不是過度遊戲化。

## 6. 不做事項

- 不把原始七步話術整包塞入 prompt。
- 不新增「PUA」「DHV」「推拉」等使用者可見技術詞。
- 不把女性角色設計成可被分數直接操控的獎勵物。
- 不把 85+ 關係分數解讀成無界線親密；高分只代表更像熟人 / 曖昧對象 / 女友語氣，仍需合理場景與同意。
- 不在第一批做 Learning tab 大改版。
- 不在這份文件階段修改 Edge Function 或 Flutter。

## 7. 待 Eric 決策

2. `可進可退` 是否可作為使用者可見詞？建議先不要，改用「低壓選項」。
3. 第一批是否先做 Batch A + Batch B？這是最快能在真機感覺到差異的組合。

## 8. 建議結論

這份資料最值得整合的不是話術，而是三個底層能力：

- 會讓話題有來回：聊她、聊我、聊我們。
- 會讓自己有畫面：狀態 + 感受、生活樣本。
- 會讓升溫有安全感：低壓、可退、順勢邀約。

先把這三件事做進 Opener 與 follow-up，再逐步接到 analyze-chat 與 AI 實戰練習室，會比一次大改所有 prompt 更穩。
