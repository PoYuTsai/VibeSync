# Memory Coach Spec 4 Draft: Coach Action Loop v1

> Status: rough draft, not ready for implementation
> Date: 2026-04-30
> Depends on: Spec 1 / Spec 2, and ideally Spec 3
> Parent roadmap: `docs/plans/2026-04-30-vibesync-memory-coach-roadmap.md`

## 1. Goal

Turn VibeSync from an analysis report into a coaching loop.

The user should not only see:

```text
趁熱推進見面
```

They should receive a concrete practice step, such as:

```text
今天練習低壓模糊邀約
```

## 2. Product Direction

Layer 1 AI answers when asked.

Layer 2+ AI should actively help the user review and practice.

Spec 4 is the first version of that second-layer coaching behavior, but it should stay non-agentic:

- No push notification in v1.
- No calendar / restaurant booking.
- No automatic external actions.
- No hidden Cyrano behavior.

## 3. Candidate Coach Actions

### Soft Invite

Use when:

- Heat is high enough.
- Conversation has shared activity / place / interest.
- The AI says "趁熱推進見面".

Card:

```text
下一步：練習低壓邀約
先不要直接敲死時間，試著丟一個輕鬆方向。

範例：
「這家感覺你會喜歡，改天可以一起去踩點」
```

### Lower Pressure Reply

Use when:

- User seems anxious or over-explaining.
- Partner response is mild / uncertain.

Card:

```text
下一步：降低壓力
這段先不要追問太多，回短一點，留空間給對方接。
```

### Explain Less

Use when:

- User sends long defensive messages.
- User profile includes `explainLess`.

Card:

```text
下一步：少解釋一點
這次把回覆控制在 20 字內，只回一個重點。
```

### Build Closeness

Use when:

- Conversation is warm but shallow.

Card:

```text
下一步：加深一點
不要只接事件，問一個感受或偏好的問題。
```

## 4. Domain Knowledge Rules

### Soft Invite

Adopt as a healthy low-pressure technique:

- No fixed time/place at first.
- Test willingness.
- If green light, move to specific invite.
- If yellow/red light, do not pressure.

### Push/Pull

Do not adopt manipulative push-pull framing.

Healthy rewrite:

- Keep your own rhythm.
- Do not chase.
- Leave space.
- Do not create anxiety intentionally.

### Photo/Profile Analysis

Rule:

```text
Photos provide conversation clues, not personality diagnosis.
```

Allowed:

- Extract observable interests and context.
- Suggest open-ended questions.
- Suggest safe topic directions.

Not allowed:

- Diagnose personality, morality, mental state, or relationship value from photos.
- Encourage stalking or invasive OSINT.

## 5. Coach Loop Shape

v1 can be simple:

1. Show one Coach Action Card after analysis.
2. User can copy / use suggested wording.
3. Later, user adds the next conversation.
4. AI can infer whether the move worked from the next screenshot.

Do not build explicit task completion tracking in v1 unless the design feels too passive.

## 6. Future Review Loop

Potential v2:

```text
上次建議你試低壓邀約，對方回應偏綠燈。
這次可以把時間範圍縮小一點。
```

This requires memory of prior action cards and should not be included in v1 unless explicitly scoped.

## 7. Non-Goals

- No push notifications.
- No external calendar / booking integrations.
- No automatic message sending.
- No manipulative PUA tactics.
- No full curriculum engine.
- No scoring the user harshly.
