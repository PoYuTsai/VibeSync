# VibeSync 隱私權政策

最後更新：2026-04-05
生效日期：2026-04-05

## 1. 適用範圍

本隱私權政策說明 VibeSync 如何蒐集、使用、分享、保存與刪除資料。

VibeSync 是一款聊天分析與回覆建議工具。服務包含：

- 帳號登入與驗證
- 對話內容分析與回覆建議
- 截圖辨識與對話匯入
- 訂閱與額度管理
- 反饋與客服支援

如果您使用 VibeSync，即表示您理解並同意本政策所述的資料處理方式。

## 2. 我們處理哪些資料

### 2.1 您直接提供的資料

| 類型 | 例子 | 主要用途 |
|------|------|----------|
| 帳號資料 | Email、登入方式、驗證相關資料 | 建立帳號、登入、找回密碼、刪除帳號 |
| 對話內容 | 您手動輸入或貼上的聊天內容 | 產生分析結果與回覆建議 |
| 截圖內容 | 您主動上傳的聊天截圖 | OCR 辨識、轉換為對話內容、產生分析結果 |
| 個人化設定 | 例如聊天目標、互動背景、選填偏好 | 調整 AI 分析與建議品質 |
| 反饋內容 | 評分、留言、對話片段、AI 回覆片段 | 改善品質、處理問題、客服支援 |

### 2.2 系統與營運資料

| 類型 | 例子 | 主要用途 |
|------|------|----------|
| 訂閱與計費資料 | tier、產品 ID、訂閱事件、restore/transfer 狀態 | 驗證方案權限與同步訂閱狀態 |
| AI 呼叫日誌 | 模型、token、延遲、狀態、部分已遮罩的 request/response metadata | 成本追蹤、錯誤排查、服務穩定性 |
| 診斷資料 | app version、build number、平台、部分已遮罩 email、錯誤碼、錯誤訊息 | 登入與驗證問題排查 |
| 裝置與請求資訊 | 平台、請求 headers、驗證 token | 驗證身分、服務安全、除錯 |

## 3. 我們如何使用這些資料

我們會將資料用於以下目的：

- 提供登入、帳號管理、找回密碼與刪除帳號功能
- 在您主動請求時，產生對話分析、回覆建議與截圖辨識結果
- 驗證訂閱、同步方案權限、處理 restore 與 billing 狀態
- 處理客服、反饋與產品品質問題
- 監控成本、除錯、避免濫用與維持服務安全
- 履行法律義務、處理爭議與防止濫用

## 4. 對話內容、截圖與 AI 處理的特別說明

這是 VibeSync 最重要的隱私說明。

- 您的對話內容與截圖預設是 local-first，主要保存在您的裝置上。
- 只有在您主動點擊分析、截圖辨識或相關 AI 功能時，完成該次請求所需的內容才會被送到我們的後端處理流程與第三方 AI 供應商。
- 我們不把原始對話內容當作長期營運資料庫中的核心產品資料來保存。
- 我們不會使用您的原始對話內容訓練我們自己的模型。

但您也應理解：

- AI 請求在傳輸與處理期間，仍會經過我們的後端與第三方處理者。
- 如果您主動提交負評或客服反饋，您附上的對話片段、留言或 AI 回覆片段可能會被保存，用於除錯與支援。

## 5. 第三方服務與資料分享

我們會在提供服務所必要的範圍內，與以下第三方服務分享資料：

| 第三方 | 目的 | 可能接觸的資料 |
|--------|------|----------------|
| Supabase | 帳號、資料庫、Edge Functions、後端處理 | 帳號資料、訂閱資料、診斷資料、營運日誌 |
| Anthropic | AI 分析與 OCR 相關處理 | 您主動送出的對話內容、截圖與相關上下文 |
| RevenueCat | 訂閱驗證與 entitlement 同步 | App user ID、產品 ID、訂閱事件 |
| Apple / Google | 平台登入、支付與商店流程 | 平台要求的帳號與交易資料 |

我們不會在與本服務無關的情況下，出售您的個人資料。

## 6. 資料保留與刪除

Apple 的 App Review Guidelines 要求我們清楚說明保留與刪除政策，因此這裡直接寫目前的真實狀態。

### 6.1 本地資料

- 對話內容、部分個人化資訊與本地快取主要保存在您的裝置。
- 您可以透過 app 內刪除對話、刪除帳號，或移除 app/清除本機資料來刪除這些內容。

### 6.2 伺服器端資料

我們會在提供、維護、保護、除錯、計費、客服支援與履行法律義務所必要的期間內保留伺服器端資料。這些資料可能包括：

- 帳號與訂閱資料
- RevenueCat / billing 同步事件
- AI 呼叫營運日誌
- 反饋資料
- 登入與驗證診斷資料

其中：

- `ai_logs` 在目前 repo 內已有 30 天清理 function 定義，但不代表所有營運資料都已統一套用自動排程。
- 反饋、訂閱事件、診斷資料與其他營運記錄，可能在服務營運、客服、資安或法規需求下保留更久。

### 6.3 Anthropic / AI 供應商保留政策

依 Anthropic 對商業產品與 Anthropic API 的官方說明：

- 預設情況下，Anthropic 不會使用商業/API 客戶的 inputs 或 outputs 來訓練模型。
- Anthropic API 的輸入與輸出，預設可能在其後端保留最長 30 天，除非使用其他具不同保留規則的服務，或另有 zero-retention agreement。

因此，當您主動使用分析或截圖辨識功能時，相關內容可能會依第三方處理者的官方政策被暫時保留。

## 7. 您的權利與可用控制

您可以：

- 修改或更新帳號資料
- 在 app 內刪除對話
- 使用 app 內的刪除帳號功能
- 來信要求協助刪除帳號或處理隱私問題

依目前服務設計，刪除帳號會嘗試刪除主要關聯伺服器端紀錄，例如：

- auth user
- subscriptions
- revenue events
- token usage
- ai_logs
- feedback
- webhook_logs

若因法規、爭議處理、資安或平台規則而必須保留部分資料，我們可能無法立即刪除全部記錄。

## 8. 兒少使用

VibeSync 目前定位為 17+ 使用者的服務。若您未滿可適用年齡，請不要使用本服務。

## 9. 政策更新

我們可能依產品變更、法規要求或營運需要更新本政策。若有重大變更，我們會在 app、網站或相關入口頁面更新版本日期。
