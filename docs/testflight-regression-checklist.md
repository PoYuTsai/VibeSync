# TestFlight 回歸檢查表

**版本用途：** 每次要發新的 TestFlight build 前後，都用這份清單快速確認核心流程沒有被最近的修正打壞。  
**目前適用階段：** v41 之後的登入 / 訂閱 / OCR 匯入密集回歸。

## 使用方式

1. 先安裝最新 TestFlight build
2. 按照下列情境逐一驗證
3. 每個情境只記錄三件事：
   - 是否通過
   - 如果失敗，卡在哪一個畫面或提示
   - 如果是 OCR，順手記 `上次 OCR 量測` 的數字

---

- [ ] Apple Sign In 可正常登入並回到首頁
- [ ] Google Sign In 可正常登入並回到首頁
- [ ] Email 註冊後能收到驗證信
- [ ] Resend verification 可正常送出

- [ ] Forgot password 冷啟動可完成

- [ ] 刪除帳號後會回到登入頁，被刪除帳號不能直接再登入

## C. OCR 主流程

### C1 正常聊天截圖

- [ ] 1 張正常雙人聊天截圖可辨識成功
- [ ] 2-3 張連續聊天截圖可按順序匯入
- [ ] 匯入目前對話後，`立即分析` 可正常工作

### C2 匯入模式

- [ ] 最新續聊截圖可選 `加入目前對話`
- [ ] 另一個人的截圖，預設偏向 `另存成新對話`
- [ ] 建立新對話後，`前往新對話` 會進到新 thread

### C3 LINE / 繁中 / 長截圖

- [ ] LINE 有「回覆」引用框的截圖，不會把引用預覽多拆成一則新訊息
- [ ] 長篇繁體中文截圖，主要內容不會大量簡化、亂猜或漏行
- [ ] 保留標題列與完整泡泡時，辨識率明顯優於裁太多的圖

### C4 邊界情境

- [ ] 社群貼文 / 留言串截圖會被拒絕，不會匯入對話
- [ ] 模糊或截到一半的聊天圖，會走 `低信心` 提示而不是默默匯入
- [ ] OCR 取消後不會跳回晚到結果
- [ ] 重新選圖後，不會殘留上一批的 warning / preview / 結果

## D. 手動輸入 / 分析

- [ ] 最後一則是她，可正常分析
- [ ] 最後一則是我，可正常分析她上一則回覆

## F. 通過門檻

可以發給伙伴測試，至少要滿足：

- [ ] A 區全部通過
- [ ] B 區全部通過
- [ ] C1 / C2 全部通過
- [ ] C3 至少 2/3 通過，且沒有嚴重誤匯入

2. 失敗類型
3. 有沒有 warning
4. 有沒有走錯匯入模式
5. OCR 量測數字
