## 訊息計算邏輯（ADR #19 r3，2026-06-11 起）

> 逐則計費（換行分割、每 200 字 1 則）已退役。現制 = **全對話字數合併計費**：
> 手動輸入文字 + 截圖 OCR 文字統一走同一條公式；繼續分析只算**新增字數差**。
> 完整規格與 Codex 把關紀錄見 `docs/decisions.md` ADR #19。

### 計費表（整數閉區間，作用對象 = 本次計費字數）

| 本次計費字數 | 扣費 | 備註 |
|------|------|------|
| 1~40 字 | 1 則 | 每按一次分析最少扣 1 則 |
| 41~400 字 | `ceil(字數/40)` = 2~10 則 | |
| 401~2000 字 | 一律 10 則 | 緩衝帶（soft cap） |
| 2001~4000 字 | **固定 20 則，需用戶確認後才扣** | 確認框顯示精確扣費 |
| 4001+ 字 | 拒絕分析「內容過長，請分批分析」 | **不扣費** |

- **計費字數** = sanitized + trim 後的訊息內容，UTF-16 code units（JS/Dart `String.length` 一致），引用預覽（quotedReplyPreview）不計費。
- **增量計費**：繼續分析只扣「新增字數」的對應則數；soft cap 每次分析各自算，不跨次累計。
- **>2000 字確認**：client 預警 + server 守門兩層；確認綁定 payload hash + idempotency（重送/雙送絕不重扣 20）。拆兩批（各 ≤2000）= 10+10 = 20，一次過也是 20，怎麼選都不吃虧。
- **舊版 App**（未送 `billingProtocolVersion: 3`）：>2000 字往便宜方向收 10 則（user-safe），4001+ 同樣拒絕。

### Essential「我幫你修」例外（ADR #22）

- 草稿潤飾每次**成功產生可用修句固定扣 1 則**，不按帶入的聊天脈絡字數增加。
- AI、格式、扣費前網路失敗或確定未提交的扣費確認失敗不扣；扣費回應中斷時以同一 UUID 安全重試，不會重複扣費。測試帳號維持免扣。
- 同一請求因網路重送時不重複扣費，並回傳第一次成功結果。
- 同一請求的免費重播期限為 7 天；逾期視為新請求。`pg_cron` 每小時清除逾期 live-table 列（最晚約 7 天 + 1 小時）；備份／PITR 副本依 Supabase 供應商週期處理。
- 重播 ledger 欄位只存 AI 產生的潤飾句與理由，不另存原始草稿、完整對話輸入、usage 或 telemetry；生成文字仍可能反映草稿、姓名與對話內容。App 以當次 hash 綁定的草稿重建畫面。
- 既有輸入安全上限仍適用；超過 4000 計費字元或其他 payload hard cap 仍拒絕且不扣。
- 此例外只適用 Essential 草稿潤飾，不改一般分析、圖片分析、Opener 或 Coach 的計費。

### 邊界情況處理

| 情況 | 處理 |
|------|------|
| 空白輸入 | 提示「請輸入對話內容」，不扣額度 |
| 只有空格/換行 | 同上 |
| 超過 4000 字 | 拒絕「內容過長，請分批分析」，**不扣費**（server 守門 + client 預警） |
| 含 emoji | 正常計算，1 emoji ≈ 2 字（UTF-16） |
| 含圖片/貼圖描述 | 用戶需手動描述，按文字計算 |
| 純圖片（食物照/貼圖） | OCR 分類即擋下（`importPolicy: reject`），不進計費 |

### 各層保證分析次數（以正常 ≤2000 字分析計）

> ⚠️ pricing 文案與 App Review 說法**不得**引用以下數字而不帶「≤2000 字」前提。

| 方案 | 月額度 | 保證次數（≤2000 字） | 若全為 >2000 確認分析 |
|------|------|------|------|
| Free | 30 則 | 至少 3 次/月 | 0 次（日上限 15 < 20，自然引導分批或升級） |
| Starter | 300 則 | 至少 30 次 | 15 次 |
| Essential | 800 則 | 至少 80 次 | 40 次 |

## UI 顯示規則

### 分析前預覽（ADR #19 r3：靜態區間，不報精確值）

```
┌─────────────────────────────────────────┐
│  依對話複雜度使用 1–10 則               │
│                                         │
│  剩餘額度：288 / 300 則                 │
│  ████████████████████████░░░            │
│                                         │
│  [取消]              [確認分析]         │
└─────────────────────────────────────────┘
```

- 分析前不算精確值；分析後顯示 server 回傳的**實際消耗則數**。
- 「絕不驚訝」由區間上限 = soft cap 10 保證；唯一例外 >2000 字必先跳確認框（顯示精確「本次將扣 20 則」）才可超 10。
- **>2000 確認順序**：先檢查額度/每日上限（不足走額度不足 UI，**不出確認框**）→ 足夠才跳確認。

> **繼續對話**：只收**增量**計費（字數差），不重複計算舊對話（2026-04；ADR #19 改字數制）
> **開場救星**：**一律 3 則**（不論幾張截圖；上限仍 3 張）— 2026-05-16 改，前版「基本 3 + 每張 +2」已停用，rationale 見 `docs/decisions.md` ADR #18
