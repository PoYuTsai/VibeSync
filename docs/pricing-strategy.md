# VibeSync 定價策略 (風險最小化版)

## 核心原則

> **永遠確保：收入 > 成本 × 3**
>
> 每個方案的最大成本不超過售價的 33%

---

## 推薦方案：取消 Unlimited，改用高額度 Pro+

### 方案設計

| 方案 | 月費 (TWD) | 分析次數 | 成本上限 | 毛利率 |
|------|------------|----------|----------|--------|
| **Free** | NT$0 | 5 次/月 | $0.025 | N/A |
| **Pro** | NT$149 | 100 次/月 | $0.50 | 89% |
| **Pro+** | NT$299 | 300 次/月 | $1.50 | 84% |

### 為什麼這樣設計

```
Free:  5 次 → 讓用戶體驗，轉換付費
Pro:   100 次 → 每天約 3 次，一般用戶足夠
Pro+:  300 次 → 每天約 10 次，重度用戶也夠用
```

**沒有 Unlimited = 沒有無限風險**

---

## 備選方案：保留 Unlimited 但加上隱藏上限

如果行銷上需要「Unlimited」這個詞：

| 方案 | 顯示 | 實際上限 | 月費 |
|------|------|----------|------|
| Unlimited | 無限制* | 500 次/月 | NT$399 |

```
* 小字：「公平使用政策：每月超過 500 次將進入排隊模式」
```

### 超過 500 次的處理

1. **不是拒絕，而是降速**
2. 請求進入佇列，等待 5-10 秒才處理
3. 顯示「高峰期處理中，請稍候」

這樣：
- 正常用戶不受影響
- 濫用者覺得體驗差，自然流失
- 行銷上還是可以說「Unlimited」

## 定價心理學

### 錨定效應

```
                    ┌─────────────┐
                    │   Pro+      │  ← 錨點 (讓 Pro 看起來便宜)
                    │   NT$299    │
                    │   300 次    │
                    └─────────────┘
                          │
                          │ 「Pro+ 好貴，Pro 比較划算」
                          ▼
┌─────────────┐     ┌─────────────┐
│    Free     │     │    Pro      │  ← 目標方案 (希望最多人買)
│    NT$0     │────▶│   NT$149    │
│    5 次     │     │   100 次    │
└─────────────┘     └─────────────┘
「5 次不夠用」        「100 次剛好」
```

### 價格尾數

- NT$149 而不是 NT$150 → 心理感覺 < 150
- NT$299 而不是 NT$300 → 心理感覺 < 300

## 定價調整策略

### 上線後觀察指標

1. **轉換率** < 3% → 降價或加功能
2. **用量/人** > 80% 上限 → 上限太低，需調整
3. **成本/收入** > 30% → 成本控制有問題
4. **Churn rate** > 10%/月 → 價值不足

### 調價原則

- **只能漲價，不能降價** (傷害早期用戶)
- 新用戶用新價，舊用戶保持原價
- 每季度最多調整一次
