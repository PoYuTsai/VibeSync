# VibeSync 上架後行銷計劃

> **撰寫日期**：2026-07-07
> **目標**：正式上架後最大化「訂閱數」與「續訂率」
> **前提**：每月行銷預算 NT$1–3 萬；小團隊、創辦人親自執行為主
> **功能主打**：Analyze-Chat（截圖分析）＋ AI 實戰練習室；Coach 1:1 為第二層深度賣點
> **定價依據**：`docs/pricing-final.md`（Free 30 則/月、Starter NT$590/300 則、Essential NT$1,290/800 則）
> **定位依據**：`docs/2026-04-18-positioning-strategy-analysis.md`、`docs/app-store-strategy.md`

## 一、目標與北極星指標

### 北極星指標

**付費訂閱數（MRR）**——所有行銷動作最終都要回答「這對訂閱數有什麼貢獻」。

### 護欄指標

| 指標 | 定義 | 為什麼重要 |
|------|------|-----------|
| Free→Paid 轉換率 | 註冊後 30 天內訂閱比例 | 獲客品質與 paywall 效率 |
| 月續訂率 | 月繳用戶下期續訂比例 | 產品價值是否撐得起 NT$590/1,290 |
| 季繳占比 | 新訂閱中選季繳的比例 | 季繳 = 預付 3 個月的續訂保險 |
| D7 / D30 留存 | 下載後第 7/30 天仍活躍 | 續訂的前導指標 |
| 首日完成首次分析率 | 下載當天完成第一次截圖分析 | wow moment 觸達率 |

### 分階段目標（假設值，上線後以實際數據校準）

| 階段 | 期間 | 主要目標 | 參考指標（區間） |
|------|------|---------|----------------|
| **冷啟動** | 上架 0–30 天 | 驗證漏斗、拿到第一批真實轉換數據 | 下載 500–1,500；付費 10–40 人 |
| **調校** | 31–90 天 | 找到 1–2 個可重複的獲客管道；續訂率基線 | 下載月增 30%+；Free→Paid 3–6% |
| **放大** | 91–180 天 | 把驗證過的管道加碼；口碑/推薦開始貢獻 | MRR 月增 20%+；月續訂率 ≥70% |

> 冷啟動期的重點**不是衝量**，是把「哪個管道帶來會付費的人」搞清楚。前 90 天每一筆訂閱都要能回答「他從哪來」。

## 二、定位與訊息框架

### 核心定位（沿用定位戰略分析）

- **品類名**：AI 約會對話教練（台灣第一個）
- **差異化語**：截圖就能分析，五種風格教你回
- **利益語**：口袋裡的約會教練
- **Consumer headline**：你專屬的 AI 約會教練

### 功能主打層級

| 層級 | 功能 | 行銷角色 |
|------|------|---------|
| **主打 1** | **Analyze-Chat 截圖分析** | wow moment、最低摩擦入口。「LINE 截圖丟進來，熱度、訊號、怎麼回，5 秒看懂」 |
| **主打 2** | **AI 實戰練習室** | daily hook 兼差異化。「跟 AI 角色實戰演練，翻牌開局、已讀救援，練完再上場」 |
| 第二層 | Coach 1:1 | 深度留存賣點：「不只教你回這句，還記得你們聊到哪」 |
| 入口 | 開場白（opener） | 導流功能：從「不知道怎麼開場」把人帶進來 |

所有內容模板、ASO 截圖、KOL brief 都以兩大主打為敘事核心。

### 價格錨點（對照現行定價）

| 對比 | 價格 | 話術方向 |
|------|------|---------|
| Tinder Gold | ~NT$1,000/月 | 「約會 App 幫你配對，VibeSync 教你配對之後怎麼聊」 |
| 真人約會教練 | NT$3,000+/次起 | 「一次真人咨詢的 1/5 價格，用一整個月」 |
| Starter NT$590 | — | 「每天不到 NT$20，比一杯手搖便宜」 |
| ChatGPT 免費 | NT$0 | 攻擊點見下 |

### 對 ChatGPT 的攻擊點（最大競品）

「別再把截圖打成字問 ChatGPT 了」：

1. **截圖直接分析**——不用手動描述情境
2. **記得你們聊到哪**——不用每次重新貼上下文
3. **熱度視覺化**——不是一段模糊的文字建議
4. **五種風格**——共鳴/調情/幽默/冷讀/延展，不是一個安全牌答案
5. **可以先練再上**——練習室實戰演練，ChatGPT 沒有

### 合規與文案紅線（違反 = 直接改稿）

- ❌ PUA、把妹、套路、操控、「讓她不得不回你」等操縱框架（台灣 PUA 汙名強烈，也違反 App Review 定位）
- ❌ 保證脫單、保證約到、保證回覆率等結果承諾
- ❌ 「絕不上傳雲端」「資料永不離開裝置」等絕對隱私說法（正確版本見 `docs/app-store-strategy.md`）
- ❌ 引用「保證分析次數」（如 Starter 至少 30 次）時**必須**帶「≤2000 字」前提
- ✅ 正確框架：真誠、自我提升、溝通品質、「幫你把想說的說好」

## 三、訂閱漏斗設計

```
曝光（Dcard/PTT/Threads/IG/ASA）
  ↓                          量測：各管道觸及、點擊
下載（App Store 頁）
  ↓                          量測：頁面轉換率（ASO 負責）
首次截圖分析 wow
  ↓                          量測：首日完成首析率
  │  ← Free 首析升 Sonnet 就是為了這一刻，品質即轉換投資
練習室每日翻牌 → 養成回訪
  ↓                          量測：D7 留存、週分析次數
Free 額度逼近/耗盡（30 則/月、15 則/日）
  ↓                          量測：quota 耗盡人數、paywall 曝光
Paywall → 訂閱
  ↓                          量測：paywall→購買轉換、月/季繳占比
續訂 / 升級（Starter→Essential）
                             量測：續訂率、升級率、流失原因
```

每層的行銷責任：

| 漏斗層 | 行銷動作 |
|--------|---------|
| 曝光→下載 | 管道計劃（第四節）＋ ASO |
| 下載→首析 | onboarding 文案、App Store 截圖預告「第一步就是丟截圖」 |
| 首析→回訪 | 練習室每日翻牌（Free 1 次/日）、48h 跟進提醒 |
| 額度耗盡→付費 | paywall 文案對齊主打敘事、額度用罄時刻的升級訊息 |
| 付費→續訂 | 第五節續訂策略 |

## 四、管道計劃（月預算 NT$1–3 萬）

### 管道優先級總覽

| 優先級 | 管道 | 成本 | 角色 |
|--------|------|------|------|
| P0 | Dcard 感情板/AI 板 | 免費（時間） | 種子擴散、SEO 長尾 |
| P0 | PTT Boy-Girl | 免費（時間） | 原點市場（男性約會討論） |
| P0 | Threads 創辦人帳號 | 免費（時間） | 品牌人格、日常觸達 |
| P1 | Instagram Reels | 低（製作時間） | 視覺展示兩大主打 |
| P1 | LINE OA | 低 | 私域承接、再行銷 |
| P2 | Apple Search Ads | 付費（預算 60–70%） | 精準攔截搜尋意圖 |
| P2 | 微型 KOL 合作 | 付費（預算 30–40%） | 信任背書 |

### P0：免費有機管道（上架第一天就開跑）

**Dcard（感情板、AI 板）**

- 內容形式：**實戰對話分析文**，不硬廣。「這段對話她到底什麼意思？我用 AI 分析了一下」→ 貼熱度分析結果 → 文末自然帶到 App
- 頻率：每週 2–3 篇；輪流用「已讀不回急救」「曖昧訊號判讀」「開場白對比」題材
- 紅線：遵守板規、不開分身洗文；被問到才給 App 名，寧慢勿封

**PTT Boy-Girl**

- 同樣以分析文/心得文切入，語氣更直接（板風偏男性、實戰導向）
- 目標是討論串發酵，不是導流連結（PTT 對廣告極敏感）

**Threads 創辦人帳號**

- 定位：「對話急診室」——每天一則短內容：一段（去識別化）對話 + 一句點評 + 怎麼回會更好
- 練習室題材：「今天 AI 角色出這題，你會怎麼接？」互動型貼文，天然帶出產品
- 頻率：每日 1 則，養演算法；台灣 Threads 44%（18–24 歲）滲透率高、文字型內容成本最低

### P1：低成本內容管道

**Instagram Reels**（每週 2–3 支，模板化量產）

可重複產製的模板，全部圍繞兩大主打：

1. **截圖分析 before/after**：「他這樣回被句點 → AI 教他這樣回 → 對話活了」
2. **練習室實戰挑戰**：「AI 角色已讀你，60 秒內你會怎麼救？」→ 展示翻牌與續玩
3. **熱度計反應**：把對話丟進去看熱度分數的即時反應（低製作成本、高重看率）
4. **五風格對比**：同一句話，共鳴/調情/幽默/冷讀/延展五種回法排排站
5. **街訪/投票型**：「這句開場白幾分？」互動貼紙導流

**LINE OA**

- 角色：私域承接（Dcard/IG 導流的落點之一）＋ 再行銷（新功能、限時季繳優惠）
- 台灣 LINE 滲透率 94%+，OA 訊息開啟率遠高於 email

### P2：付費管道（月 NT$1–3 萬）

**Apple Search Ads（預算 60–70%，約 NT$0.7–2 萬/月）**

- 台灣 iOS 市佔 54–56%、iOS-first 策略下 ASA 是最精準的付費管道
- 關鍵字組：
  - 意圖詞：`聊天回覆`、`不會聊天`、`已讀不回`、`開場白`、`約會`、`曖昧`
  - 競品/場景詞：`交友軟體`、`tinder`、`聊天技巧`
- 起步日預算 NT$300–500，先跑 2 週看 CPA，只留 CPA < 首月訂閱毛利（Starter 約 NT$490）的字組

**微型 KOL（預算 30–40%，約 NT$0.3–1.2 萬/月）**

- 目標：1–5 萬粉的兩性/交友類 YouTuber、Threads/IG 創作者，每月 1–2 位
- 形式：互惠（免費 Essential）＋ 小額稿費；brief 固定為「實測截圖分析 + 練習室挑戰」兩段式
- 給每位 KOL 專屬 promo code / 專屬連結做歸因（見第七節）

**每月再平衡**：月底看各管道 CPA 與付費轉換，把下月預算往表現最好的管道傾斜。

### ASO（App Store 頁面優化，一次做好、每月微調）

- **名稱/副標**：`VibeSync - 社交溝通教練`（對齊 App Review 安全定位），副標帶「截圖分析」「回覆建議」關鍵字
- **截圖敘事順序**：① 截圖分析 wow moment（熱度計 + 訊號）② 五風格回覆 ③ 練習室翻牌實戰 ④ Coach 1:1 ⑤ 隱私承諾
- **評分請求時機**：分析成功且結果被採用（複製回覆）後才彈，不在額度用罄或錯誤後彈
- **關鍵字欄**：聊天、回覆、約會、開場白、曖昧、溝通、練習

## 五、續訂率策略

> 續訂率的地基是產品價值兌現，行銷只能放大。本節按用戶生命週期設計，部分依賴第六節產品端機制。

### 啟用期（D0–D7）：讓價值在第一週兌現

- onboarding 目標：**首日完成第一次截圖分析 + 第一次練習室翻牌**（兩大主打各體驗一次）
- 額度使用進度可視化（現有 UI 已具備），讓 Free 用戶清楚感知「我在消耗有價值的東西」
- 48h 跟進提醒（已實作）把「分析完就走」的人拉回來看後續

### 習慣期（D7–D30）：建立回訪理由

- **練習室每日翻牌是核心 daily hook**：Free 1 次/日給回訪理由；付費 3/5 次＋續玩同角色＋加購翻牌是清晰的升級階梯
- 學習專區維持內容更新節奏（Free 3 篇/日的上限本身就是回訪＋升級雙重誘因）
- Coach 1:1 作為深度用戶的黏著層：用過 Coach 的用戶預期留存顯著更高，onboarding 後期引導試用

### 轉換期：把「額度耗盡」變成訂閱時刻

- Free 額度逼近時（如剩 5 則）預告而非突襲；耗盡時 paywall 文案對齊使用情境（「你這個月分析了 X 段對話」）
- **季繳是續訂率槓桿**：paywall 預設 essential_monthly（現況），季繳選項強調折扣與「3 個月穩定練」；季繳一單 = 鎖定 3 個月留存

### 續訂期：在扣款日之前證明價值

- **到期前價值回顧**（需第六節 P1 機制）：「這個月 VibeSync 陪你分析了 X 段對話、練了 Y 場」——在續訂日前 3–5 天推送
- Starter 額度常態用罄的用戶 → 推 Essential 升級而非放任體驗變差流失
- 額度月中用罄 → 加購訊息包（第六節 P0）是「不升級也別走」的保護閥

### 挽回期：流失不是終點

- RevenueCat webhook 已有 `CANCELLATION` / `BILLING_ISSUE` 事件流入 `webhook_logs`：
  - `BILLING_ISSUE`：推送「扣款失敗，檢查付款方式」（救回非自願流失，通常占流失 20–30%）
  - `CANCELLATION`：到期前推一次挽留（取消原因一鍵問卷 + 季繳/降級選項）
- 取消原因分類累積 30 天後回頭修產品或定價敘事
- Win-back：流失 30 天後推「回來看看新功能」＋ 限時優惠（若 App Store 促銷代碼可行）

## 七、指標儀表板與營運節奏

### 數據來源（沿用 `docs/subscription-revenue-observability.md`）

- `subscriptions`：tier 分佈、額度使用（`monthly_messages_used` 可直接算 engagement）
- `revenue_events`：新訂閱、續訂、金額，按 `product_id` 分月/季繳
- `webhook_logs`（source=revenuecat）：`CANCELLATION` / `BILLING_ISSUE` 流失信號

### 週報固定看板（每週一次，30 分鐘內看完）

1. 新增下載（App Store Connect）與新增註冊
2. 首日完成首析率、D7 留存
3. 新增訂閱數（分 tier、分月/季繳）、MRR
4. Free→Paid 轉換率（30 天窗口）
5. 流失事件數與原因分佈
6. 各管道歸因：本週各管道帶來的註冊/訂閱

### 管道歸因（初期從簡）

- KOL / 活動：專屬 promo code 或專屬短連結
- 有機管道：註冊後一題式問卷「你從哪裡知道 VibeSync？」（Dcard / PTT / IG / Threads / 朋友 / 搜尋）
- ASA：App Store Connect 原生歸因

### 營運節奏

| 頻率 | 動作 |
|------|------|
| 每日 | Threads 發文；Dcard/PTT 留言區維護 |
| 每週 | 週報看板；2–3 篇 Dcard/Reels 內容；調整 ASA 出價 |
| 每月 | 管道預算再平衡；KOL 下月排程；計劃全盤檢視、更新本文件 |

## 八、風險與競品監測

| 信號 | 危險度 | 監測 | 應對 |
|------|--------|------|------|
| Rizz / Keys 推出繁中版 | 極高 | 每月搜 App Store 繁中關鍵字 | 加速在地化壁壘（LINE 情境、台灣用語、練習室角色本土化）；強打「懂台灣的約會語境」 |
